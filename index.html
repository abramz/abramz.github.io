Short Url

Short Url is a simple URL shortener. It contains three components:

  • The Front End is pretty simple, one HTML page and a few lines of JavaScript. It can be hooked up to travis-ci and Google Cloud Storage. This allows successful builds to be deployed to a bucket in Google Cloud Storage. I use gulp to run tasks, including unit tests. That means, that the code is tested on travis-ci before deployment.

  • The API is a node.js application that stores/retrieves long URLs in a MySQL database. I use express.js to run an API that will:

    • "shorten" a long URL that is POSTed to '/'
    • returns a long URL that is GETed to '/s'

    I use mocha to run unit tests, and have a Docker file so that the API can run in a container. The API can be hooked up to travis-ci and Google Container Engine. I can run tests with mocha, so that code is tested before deployment. On a sucessful build of the master branch, an updated image, identified by the build id, is sent to the Google Container Registry. Then, a rolling update is run on a Google Container Cluster.

  • The Proxy is an nginx reverse proxy that:

    • sends GET requests to '/', to the Google Cloud Storage bucket containing the front end
    • sends all other requests to the API running in a Google Container Cluster

    While there are no tests for the proxy, when a check-in is made to the master branch, a travis-ci build does kick off, and if the Docker image can be built, it has the same deployment scheme as the API.

Patch Diff - Riot API Challenge 2.0 (live) launch

For Riot's API Challenge 2.0, I worked with Ethan Pursley to develop a website that display differences between League of Legends patches 5.11 and 5.14. The release of patches 5.12, 5.13, and 5.14, brought a series of changes made to AP (ability power) items in the game. We received a data set of 400,000 match identifiers, and while the data set is not all inclusive, there are 200,000 games for each patch, we were able to display some interesting stats.

gulp-render-react

gulp-render-react is a plugin for the popular Node.js build system, gulp.js. I created this plugin so that I could pre-render React components in the build process. I wanted to do this to decrease initial load time of static web applications.
